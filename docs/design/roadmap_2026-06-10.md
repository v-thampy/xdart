# XRD Toolkit Roadmap — 2026-06-10

Two coupled repos: **`ssrl_xrd_tools`** (headless analysis/IO core) and **`xdart`** (thin Qt live-experiment
GUI over it). This is the forward-looking source of truth; per-task detail lives in the other `review/`
docs and in each repo's `CLAUDE.md`.

## North star (long-term goals)
1. **Headless-first ssrl APIs** — the analysis core is fully usable with no GUI (notebooks, scripts, future
   services). `ssrl_xrd_tools` imports **no Qt**, ever.
2. **Thin xdart** — the GUI is a thin shell over the core; anything not Qt-specific belongs in ssrl.
3. **Robustness** — fail-loud on data-writing errors, strict schema validators, correctness spines
   (live ≡ batch ≡ reload), no silent-wrong-data.
4. **Performance** — streaming, parallel reduction that scales; bounded memory; live browsing without
   freezes.
5. **Expandability** — clean seams so new data sources, analyses, and viewers slot in **additively**
   without touching the core.

## How Architecture V2 delivers on these
- **One streaming reduction spine** (`ssrl reduction.ReductionSession`: parallel worker pool + single
  writer thread, bounded in-flight, fail-loud `finish()`) → *performance* (live/batch both stream by
  default; live 2D ~27.5 s vs 59–76 s serial) and *robustness* (write failures raise, never silently
  "succeed").
- **Source + sink abstraction** (`sources/` + `open_source(spec)`; `ReductionSink` protocol with xdart
  injecting its Qt `QtNexusSink`) → *headless-first* (ssrl runs the spine with no Qt) and *expandability*
  (new sources/sinks are additive; ssrl never imports xdart).
- **N1 portable raw-source paths** (relative `source/path` + `entry/@source_base`, resolved in the ssrl
  readers with a `source_root=` override) → *robustness/portability* (a processed `.nxs` resolves its raw
  after the data moves machines).
- **xdart display layer** (Qt-free `display_logic` decision core + keyed `PanelKey`/`layout` descriptors +
  registered controllers) → *thin xdart* and *expandability* (Stitch/RSM viewers register panels; the core
  never imports them).
- **GI whole-scan grid freeze, strict writer validators, the live≡batch≡reload spine** → *robustness*.

## Strategic direction (greenfield, 2026-06-09) — NOT this release
Full detail: `greenfield_design_2026-06-09.md`. The honest finding: a from-scratch design would differ less in
*architecture* (contracts, pure display core, equivalence spine all reappear) than in *infrastructure* —
where code lives, who owns the data, what's enforced mechanically. Differences, with retrofit verdict:

1. **★ Monorepo / workspace (Vivek's chosen direction — "maintaining 2 repos is a pain").** One repo,
   `packages/xrd-core` + `packages/xdart`, one lockfile, one test invocation, one CI matrix, atomic cross-package
   commits; ordered PyPI publish becomes a script not a runbook; `review/` becomes versioned `docs/`. Deletes
   whole finding classes (C2 schema-split, C4 floor-bypass, C7 cross-repo CI). **Retrofit:** thin workspace repo
   with both as submodules + shared CI, or at minimum mirrored GitHub Actions (xdart CI checks out ssrl head;
   ssrl CI runs xdart's seam tests); move `review/` into the repo(s). Plan this as its own effort after the
   release.
2. **Boundary at data-ownership, not Qt — a headless `xrd-session` layer** owning the live scan state machine
   (start/pause/resume/stop), the single writer, save cadence, eviction, publication store; xdart becomes a pure
   *view*. Makes live≡batch≡reload a property of *construction* (one path) and "thin xdart" *structural* (no API
   to own data). **Step 1 = the deep-review S3/C2 move above** — do that with this destination in mind. The
   biggest architectural steer; QtNexusSink + ReductionSession is already two-thirds of it.
3. **One `FrameRecord`, one store** (collapse ScanFrame/LiveFrame/FrameView/FramePublication + data_1d/data_2d/
   PublicationStore). Already converging via X1/X2 — finish it; target: grep "keep both stores in sync" → nothing.
4. **Working store (zarr) + NeXus export** — separate the live append target from the archival file (kills the
   shared file-lock/h5pool/SWMR machinery). **Defer** — current locking works; revisit with Tiled/Bluesky.
5. **Schema as code** — one declarative `SCHEMA` table in core (datasets/dtypes/row-aligned flags/capability
   attrs/version) that the writer layout, reader expectations, validators, and fixtures all derive from. **High
   leverage, cheap, retrofittable** — start with a row-aligned table + the C1 version check; grow opportunistically.
6. **CI + contract tests from day one** (same as C4/C7) — cheapest counterfactual; the lettered review cycles
   were the manual equivalent.
7. **Day-one policies for new code**: domain naming (no `Live*`/two `Scan` classes); xarray as the result-path
   currency; one coalescing idiom (throttle); `None` as the only missing-value sentinel; strictness flags
   default-loud for headless, GUI opts into graceful degradation.

## Pre-ship sweep trade-offs (Jun 10, deferred to post-release — none block default path)
From the final pre-merge sweep (CC). All post-release; default streaming workflow unaffected.
- **M4 — Reintegrate-All RAM (HIGH post-release perf).** All N frames stay resident during a reintegrate-all
  (~10 GB on the 651-frame Eiger) — can OOM a small machine. Fix: periodic replace-saves inside the reintegrate
  loop so eviction can run (a save-cadence change on the recovery path; do carefully).
- **data_1d thumbnails (~262 KB/frame, ~3 GB/10k).** Functional (Raw preview reads them); capping changes
  old-frame preview behavior. Per-scan bounded. Decide a cap vs on-demand rehydrate post-release.
- **PublicationStore unbounded metadata (low).** Cake's only render source; capping could blank old-frame
  cake re-display. Post-release.
- **Serial-fallback (`XDART_LIVE_EXECUTION=serial`) + Int 1D (XYE) writes ZERO output** — buffer never
  flushes. NON-default opt-in only (default streaming + true-live both write XYE fine). Silent-data-loss on
  that escape-hatch path → make it **fail loud (warn/refuse)** or fix the flush before anyone relies on it.
- Refuted in this sweep (documented, do NOT "re-fix"): per-run wrangler-thread leak, set_wrangler
  accumulation, H5FilePool race, GI-2D freeze hole — all traced to unreachable paths.
